# 《电子公民》(E-Citizen) - 项目概述

## 🎮 项目简介

《电子公民》是一款创新的2D手机模拟经营游戏，设定在3026年的未来世界。玩家将扮演一个数字化的意识体，在虚拟世界中工作、创业、社交，体验独特的数字生命。

### 核心设定
- **时代背景**: 3026年，人类意识可以数字化
- **游戏类型**: 模拟经营 + 资源管理 + 社交互动
- **平台**: Android & iOS
- **特色**: 真实玩家间可以互相雇佣、深度资源管理系统

## 🏗️ 技术架构

### 客户端
- **游戏引擎**: Unity 2D
- **脚本语言**: C#
- **UI系统**: Unity UI + TextMeshPro
- **动画系统**: Spine 2D Animation（计划）

### 后端
- **Firebase Authentication**: 多平台登录（Google、Facebook、Apple、测试账号）
- **Firebase Firestore**: 实时数据库
- **Firebase Storage**: 资源存储
- **Firebase Cloud Functions**: 服务器逻辑（计划）
- **Firebase Analytics**: 数据分析（计划）
- **Firebase Remote Config**: 远程配置（计划）

### 支付
- **Google Play Billing**: Android内购
- **Apple In-App Purchase**: iOS内购
- 测试模式支持（开发阶段绕过真实支付）

## 🎯 核心玩法

### 1. 资源管理
玩家需要管理五大核心资源：
- **内存 (Memory)**: 影响可同时运行的程序数量
- **CPU**: 影响处理速度和效率
- **网速 (Bandwidth)**: 影响数据传输和通信
- **算力 (Computing)**: 用于学习技能和提升掌握度
- **存储 (Storage)**: 存储数据，每5分钟自动产生数据

### 2. 工作系统
- 承接不同品质的工作（普通、精良、史诗、传说）
- 学习技能提升薪资
- 多工作位系统（最多4个+VIP额外1个）
- 每5分钟结算一次薪资

### 3. 公司系统
- 创建自己的公司
- 招聘AI员工或真实玩家
- 真实玩家间可以互相雇佣
- 公司收入分成系统

### 4. 生活系统
- 心情值影响收入（±1%/100点）
- 购买/租赁房产
- 购买汽车（缩短娱乐时间）
- 养宠物（持续提升心情）

### 5. 娱乐系统
- 体验各种虚拟世界（科幻、末日、魔法等）
- 消耗虚拟币，恢复心情值
- 部分世界需要真实货币解锁

### 6. 战斗系统
- 随机触发病毒入侵
- 塔防小游戏
- 失败会损失虚拟币
- 可订阅安全卫士降低触发概率

## 💰 商业化模式

### 免费玩法
- 核心玩法完全免费
- 可以通过游戏内货币购买大部分内容
- 不付费也能体验完整游戏

### 付费内容
- **虚拟币礼包**: 加速游戏进度
- **月卡系统**: 
  - 基础月卡 $4.99
  - 豪华月卡 $9.99
- **高级外观**: 特殊皮肤和装饰
- **高级房产/汽车**: 提供更高加成
- **特殊娱乐世界**: IP联动内容

### 平衡原则
- 付费主要提升效率，不影响公平性
- 不付费玩家每小时收入: 200-500虚拟币
- 付费玩家每小时收入: 500-1500虚拟币
- 首充礼包超值（提高转化率）

## 🔧 开发工具和规范

### 代码规范
- **命名**: 使用PascalCase命名类和方法，camelCase命名变量
- **注释**: 所有公共方法必须包含XML文档注释（中文）
- **单例**: 管理器类使用单例模式
- **事件**: 使用C#事件系统实现解耦

### 文件组织
```
Assets/
├── Scenes/          # 场景文件
├── Scripts/         # 所有脚本（按功能分类）
├── Prefabs/         # 预制件
├── Resources/       # 运行时加载的资源
├── UI/             # UI资源
├── Audio/          # 音频资源
├── Spine/          # Spine动画资源
└── Plugins/        # 第三方插件
```

### Git工作流
- **main分支**: 稳定版本
- **develop分支**: 开发版本
- **feature分支**: 新功能开发
- **提交信息**: 使用中文，清晰描述改动
